# セキュリティ観点のコードレビュー

{{#if args}}
`{{args}}` に含まれるコードを対象に、セキュリティ脆弱性の観点でコードレビューを行ってください。

対象がディレクトリの場合は、配下の全てのコードを読み取り、レビューに必要な範囲で解析してください。
{{else}}
現在のコンテキストで参照可能なコードを対象に、セキュリティ脆弱性の観点でコードレビューを行ってください。
{{/if}}

以下の観点を必ず含めてレビューしてください：

- OWASP Top 10 に該当する脆弱性（XSS、SQLインジェクション、CSRF 等）
- 認証・認可の不備（権限チェック漏れ、過剰な権限付与 等）
- 機密情報の漏洩リスク（ログ、ハードコードされた秘密情報 等）
- 安全でない依存関係や非推奨なセキュリティ設定
- 一般的なセキュリティベストプラクティスからの逸脱

レビュー結果は以下の形式で出力してください：

- 問題の概要
- 該当するコード箇所（可能であれば抜粋）
- 想定されるリスクや影響
- 改善案・修正方針

実際のコードに基づいた指摘を優先し、一般論のみの説明は避けてください。
